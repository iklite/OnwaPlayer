Code of Conduct
OnwaPlayer
Our Commitment
We are committed to making OnwaPlayer an open, respectful, and collaborative project.
Everyone participating in this project is expected to contribute in a constructive and professional manner.
This applies to all contributors, users, and maintainers.
Expected Behavior
All participants are expected to:
Be respectful and considerate in all communication
Focus on technical improvement and constructive feedback
Respect differing opinions and implementation approaches
Provide clear, helpful, and actionable feedback in issues and pull requests
Be patient with contributors of different experience levels
Unacceptable Behavior
The following behaviors are not allowed:
Harassment, insults, or discriminatory language of any kind
Personal attacks or disrespectful criticism
Spamming issues or pull requests without meaningful content
Deliberately sabotaging code, reviews, or discussions
Repeatedly ignoring project guidelines or maintainer feedback
Attempting to re-upload or misrepresent the project as your own original work
Scope
This Code of Conduct applies to:
GitHub issues
Pull requests
Code reviews
Discussions or community interactions related to the project
Project Philosophy
OnwaPlayer is built to remain:
Open-source under the MIT License
Lightweight and performance-focused
Modular and extensible through contributions
Contributors are encouraged to improve the project, reuse their own work elsewhere, and build derivative systems—so long as proper attribution and licensing are respected.
Enforcement
Project maintainers reserve the right to:
Request changes or clarification on contributions
Reject or close inappropriate pull requests
Temporarily or permanently restrict participation in the project if necessary
Decisions will always aim to be fair, consistent, and focused on maintaining project quality.
Attribution & Respect
All contributors must:
Respect the original authorship of the project
Preserve licensing and attribution requirements (MIT License)
Avoid publishing the project verbatim under a different name without meaningful modification
Contact
For concerns related to behavior or moderation, open a GitHub issue or contact the maintainer via the repository.
