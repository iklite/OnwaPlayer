Contributing to OnwaPlayer
Thank you for your interest in contributing to OnwaPlayer.
OnwaPlayer
This project is an Android media player built with native MediaPlayer, custom UI components, and peer-to-peer WiFi Direct media sharing features. Contributions are welcome as long as they improve quality, stability, or functionality.
📌 Contribution Goals
We are primarily looking for contributions that improve:
Playback stability and performance
UI/UX refinement and responsiveness
Waveform seek bar accuracy and smoothness
WiFi Direct sharing and playback synchronization
Memory efficiency and battery optimization
Code clarity and maintainability
🧱 Project Context
Key custom components include:
IkVideoView — custom video playback view
IkBeautifulDialog — custom dialog system
Waveform seek bar — Canvas-based media scrubbing UI
Logging utility — logs to Logcat and Downloads folder
WiFi Direct module — peer-to-peer media sharing and sync playback
Please take time to understand these components before making changes.
🚀 How to Contribute
Fork the repository:
Bash
https://github.com/iklite/OnwaPlayer
Create a feature branch:
Bash
git checkout -b feature/your-feature-name
Make your changes:
Keep changes focused and minimal per PR
Avoid unrelated refactoring
Test thoroughly:
Ensure playback still works correctly
Check WiFi Direct behavior if modified
Verify UI responsiveness on API 23+
Submit a pull request:
Clearly describe what you changed and why
Include screenshots or logs if relevant
📱 Coding Guidelines
Use clear, readable Java code (no unnecessary obfuscation)
Avoid introducing heavy dependencies unless justified
Respect existing architecture and custom components
Maintain backward compatibility (API 23–35+)
Do not break existing playback or sharing behavior
⚠️ Important Usage & Distribution Rules
By contributing to this project, you agree that:
Your contributions may be used in this project under the MIT License.
You are free to reuse your own contributions in other projects.
Any derivative work you create should also remain open source where possible.
You must not re-publish this project verbatim as your own standalone product without meaningful modification or originality.
Proper attribution to the original project must always be preserved.
This is to ensure the project remains open, fair, and not duplicated without contribution.
🤝 Community Expectations
Be respectful and constructive in discussions
Focus on improving the project, not rewriting it unnecessarily
Collaborate rather than overwrite existing design decisions without discussion
Keep PRs clean and reviewable
